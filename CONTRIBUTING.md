# Contributing

Thanks for improving WebP Converter. Keep changes small, accessible, dependency-light, and compatible with static hosting.

## Before opening a pull request

1. Create an issue for significant feature or UX changes.
2. Run `npm run build`.
3. Test keyboard-only upload, conversion, download, a small screen, and a screen reader status announcement where possible.
4. Do not add tracking, ad code, remote fonts, or conversion limits without maintainer discussion.
5. Ensure format pages have unique, accurate content rather than mechanically rewritten text.

## Development principles

- Process files locally unless a feature explicitly informs and obtains consent from the user.
- Avoid paid infrastructure dependencies for core functionality.
- Use semantic HTML and preserve visible focus states.
- Keep external scripts optional and documented.

## Pull request format

Explain the user problem, solution, test evidence, accessibility/SEO effect, and any privacy impact. Include before/after screenshots for visual changes.
