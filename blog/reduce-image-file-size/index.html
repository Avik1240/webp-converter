Image optimization

How to Reduce Image File Size Without Losing Quality

Images are usually the heaviest thing on a web page. These five steps cut image weight by 50–80% with no visible quality loss — no paid software required.

1. Resize to the dimensions you actually display

The most common mistake is uploading a 4000-pixel photo into a 800-pixel content column. The browser downloads every pixel, then throws three quarters of them away. Before anything else, scale the image to the largest size it will ever be displayed at: roughly 1600–2000px wide for full-width heroes, 800–1200px for in-content images, and 400px for thumbnails. This step alone often cuts file size by more than half.

2. Convert to WebP

WebP compresses photographic images 25–35% smaller than JPEG and often dramatically smaller than PNG, at the same visual quality. Every modern browser supports it. Use our free image to WebP converter — it handles JPG, PNG, GIF, and more, converts entirely in your browser, and shows the size reduction per file. For a detailed comparison, see WebP vs JPEG and WebP vs PNG.

3. Tune the quality setting — don't just accept the default

Quality 100 is almost never worth it: the file can be two to three times larger than quality 85 with no difference you can see. Start at 85, look at the result at 100% zoom, and step down until you notice degradation, then go one step back up. Photos with soft backgrounds tolerate 70–75; images with fine text or sharp edges may need 90.

4. Strip metadata

Cameras and phones embed EXIF data — camera model, GPS coordinates, edit history, even a second thumbnail copy of the image. It adds weight and can leak location data. Canvas-based conversion (which this site's converter uses) drops metadata automatically as a side effect of re-encoding, so step 2 handles this for you.

5. Lazy-load everything below the fold

Even perfectly compressed images slow the first paint if the browser fetches thirty of them upfront. Add loading="lazy" to every <img> that is not visible on initial load. It's a single attribute, supported everywhere, and it defers off-screen downloads until the user scrolls near them.

How much does this matter for SEO?

Directly. Largest Contentful Paint — usually your hero image — is a Core Web Vitals metric that feeds Google's ranking systems. Pages that ship lighter images paint faster, hold mobile visitors better, and crawl more efficiently. Image optimization is the highest-leverage performance work most sites can do.

Frequently asked questions

How can I reduce image size without losing quality?

Resize to display dimensions, convert to WebP, and export at quality 80–85. Together these typically cut 50–80% of file weight with no visible loss.

What is the best image size for a website?

The largest size the image is actually displayed at: 1600–2000px for full-width heroes, 800–1200px for content images, ~400px for thumbnails.

Does reducing image size affect SEO?

Positively — smaller images improve Largest Contentful Paint, a Core Web Vitals ranking signal, and reduce mobile bounce rates.
